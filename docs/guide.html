Installation

Step 1: Install via Composer

composer require rbac-suite/omni-access

Step 2: Publish Configuration & Migrations

# Publish config file
php artisan vendor:publish --tag=omni-access-config

# Publish migrations
php artisan vendor:publish --tag=omni-access-migrations

# Publish views (optional - for custom unauthorized page)
php artisan vendor:publish --tag=omni-access-views

# Or publish everything at once
php artisan vendor:publish --provider="RbacSuite\OmniAccess\OmniAccessServiceProvider"

Step 3: Run Migrations

php artisan migrate

Step 4: Add Trait to User Model

<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;
use RbacSuite\OmniAccess\Traits\HasRoles;

class User extends Authenticatable
{
    use HasRoles;

    // Optional: Set custom guard for this model
    protected string $guard_name = 'web';
    
    // ... rest of your model
}
You're ready to go!

OmniAccess is now installed and ready to use. Continue reading to learn how to configure and use it.

Configuration

The configuration file is located at config/omni-access.php. Here's a complete overview:

Basic Configuration

<?php

return [
    /*
    |--------------------------------------------------------------------------
    | User Model
    |--------------------------------------------------------------------------
    | The user model class that will have roles and permissions.
    */
    'user_model' => App\Models\User::class,

    /*
    |--------------------------------------------------------------------------
    | Primary Key Types
    |--------------------------------------------------------------------------
    | Supported: "bigint", "int", "uuid"
    */
    'primary_key_type' => 'bigint',
    'user_primary_key_type' => 'bigint',

    /*
    |--------------------------------------------------------------------------
    | Super Admin Role
    |--------------------------------------------------------------------------
    | Users with this role will bypass all permission checks.
    | Set to null to disable super admin functionality.
    */
    'super_admin_role' => 'super-admin',
];

Guards Configuration

'guards' => [
    // Default guard (null = use Laravel's default)
    'default' => null,
    
    // Available guards for roles/permissions
    'available' => ['web', 'api', 'admin'],
],

Cache Configuration

'cache' => [
    // Enable/disable caching
    'enabled' => env('OMNI_ACCESS_CACHE_ENABLED', true),
    
    // Cache TTL in minutes
    'expiration_time' => env('OMNI_ACCESS_CACHE_TTL', 60 * 24),
    
    // Cache key prefix
    'key_prefix' => 'omni_access',
    
    // Cache store (null = default store)
    'store' => env('OMNI_ACCESS_CACHE_STORE', null),
],

Middleware Configuration

'middleware' => [
    // Auto-register middleware aliases
    'register' => true,
    
    // Middleware aliases
    'aliases' => [
        'role' => \RbacSuite\OmniAccess\Middleware\RoleMiddleware::class,
        'permission' => \RbacSuite\OmniAccess\Middleware\PermissionMiddleware::class,
        'role_or_permission' => \RbacSuite\OmniAccess\Middleware\RoleOrPermissionMiddleware::class,
    ],

    // Validate if guard exists
    'validate_guard' => true,
    
    // If true, role's guard must match exactly
    'strict_guard' => false,

    'unauthorized' => [
        // Response type: 'auto', 'json', 'view', 'redirect', 'abort'
        'response_type' => 'auto',
        
        // HTTP status codes
        'status_code' => [
            'unauthenticated' => 401,
            'forbidden' => 403,
        ],

        // Custom messages
        'messages' => [
            'unauthenticated' => 'You must be logged in.',
            'forbidden_role' => 'You do not have the required role.',
            'forbidden_permission' => 'You do not have the required permission.',
        ],

        // View configuration
        'view' => [
            'name' => 'errors.unauthorized',
        ],

        // Redirect configuration
        'redirect' => [
            'url' => '/login',
            'with_message' => true,
            'message_key' => 'error',
        ],

        // JSON configuration
        'json' => [
            'include_required' => false,
            'include_user_roles' => false,
        ],
    ],
],

Basic Setup

Creating Your First Roles and Permissions

use RbacSuite\OmniAccess\Models\Role;
use RbacSuite\OmniAccess\Models\Permission;
use RbacSuite\OmniAccess\Models\Group;

// Create permission groups
$usersGroup = Group::create([
    'name' => 'User Management',
    'slug' => 'users',
    'sort_order' => 1,
]);

$postsGroup = Group::create([
    'name' => 'Post Management', 
    'slug' => 'posts',
    'sort_order' => 2,
]);

// Create permissions
Permission::create(['name' => 'View Users', 'slug' => 'users.view', 'group_id' => $usersGroup->id]);
Permission::create(['name' => 'Create Users', 'slug' => 'users.create', 'group_id' => $usersGroup->id]);
Permission::create(['name' => 'Edit Users', 'slug' => 'users.edit', 'group_id' => $usersGroup->id]);
Permission::create(['name' => 'Delete Users', 'slug' => 'users.delete', 'group_id' => $usersGroup->id]);

Permission::create(['name' => 'View Posts', 'slug' => 'posts.view', 'group_id' => $postsGroup->id]);
Permission::create(['name' => 'Create Posts', 'slug' => 'posts.create', 'group_id' => $postsGroup->id]);
Permission::create(['name' => 'Edit Posts', 'slug' => 'posts.edit', 'group_id' => $postsGroup->id]);
Permission::create(['name' => 'Delete Posts', 'slug' => 'posts.delete', 'group_id' => $postsGroup->id]);

// Create roles
$superAdmin = Role::create([
    'name' => 'Super Administrator',
    'slug' => 'super-admin',
    'description' => 'Full system access',
]);

$admin = Role::create([
    'name' => 'Administrator',
    'slug' => 'admin',
    'description' => 'Administrative access',
]);

$editor = Role::create([
    'name' => 'Editor',
    'slug' => 'editor',
    'description' => 'Can manage posts',
]);

// Assign permissions to roles
$admin->givePermissionTo('users.view', 'users.create', 'users.edit');
$admin->givePermissionTo('posts.view', 'posts.create', 'posts.edit', 'posts.delete');

$editor->givePermissionTo('posts.view', 'posts.create', 'posts.edit');

// Assign roles to users
$user = User::find(1);
$user->assignRole('super-admin');

Managing Roles

Create Role

use RbacSuite\OmniAccess\Models\Role;

// Simple creation (slug auto-generated from name)
$role = Role::create(['name' => 'Manager']);
// slug will be 'manager'

// With all options
$role = Role::create([
    'name' => 'Content Manager',
    'slug' => 'content-manager', // Custom slug
    'description' => 'Manages all content',
    'guard_name' => 'web',
    'is_default' => false,
]);

Find Role

// By ID
$role = Role::find(1);

// By slug
$role = Role::where('slug', 'admin')->first();

// Using cached method (recommended)
$role = Role::findBySlugCached('admin');

// Get all roles
$roles = Role::all();

// Get all cached (recommended for performance)
$roles = Role::getAllCached();

// Get default role
$defaultRole = Role::getDefaultRole();

Update Role

$role = Role::where('slug', 'editor')->first();

$role->update([
    'name' => 'Content Editor',
    'description' => 'Updated description',
]);

// Note: slug is immutable by default (configurable)

Delete Role

$role = Role::where('slug', 'old-role')->first();

// Soft delete
$role->delete();

// Force delete (permanently removes)
$role->forceDelete();

// Restore soft deleted
$role->restore();

// Get soft deleted roles
$trashedRoles = Role::onlyTrashed()->get();

Role Permissions

$role = Role::where('slug', 'editor')->first();

// Give single permission
$role->givePermissionTo('posts.create');

// Give multiple permissions
$role->givePermissionTo('posts.create', 'posts.edit', 'posts.view');

// Give permissions using array
$role->givePermissionTo(['posts.create', 'posts.edit']);

// Give permission using model
$permission = Permission::where('slug', 'posts.delete')->first();
$role->givePermissionTo($permission);

// Sync permissions (replace all existing)
$role->syncPermissions('posts.view', 'posts.edit');
$role->syncPermissions([]); // Remove all permissions

// Revoke permission
$role->revokePermissionTo('posts.delete');

// Check if role has permission
if ($role->hasPermission('posts.create')) {
    // Role has this permission
}

// Get all role permissions (cached)
$permissions = $role->getCachedPermissions();

Assigning Roles to Users

Assign Roles

$user = User::find(1);

// Assign by slug
$user->assignRole('admin');

// Assign by name
$user->assignRole('Administrator');

// Assign by model
$adminRole = Role::where('slug', 'admin')->first();
$user->assignRole($adminRole);

// Assign multiple roles
$user->assignRole('editor', 'author');

// Assign multiple roles (array)
$user->assignRole(['editor', 'author']);

// Assign multiple roles (mixed)
$editorRole = Role::where('slug', 'editor')->first();
$user->assignRole('admin', $editorRole, 'author');

Remove Roles

$user = User::find(1);

// Remove single role
$user->removeRole('editor');

// Remove multiple roles
$user->removeRole('editor', 'author');

// Remove using model
$role = Role::where('slug', 'editor')->first();
$user->removeRole($role);

Sync Roles

$user = User::find(1);

// Replace all roles with new ones
$user->syncRoles('admin', 'editor');

// Replace with array
$user->syncRoles(['subscriber']);

// Remove all roles
$user->syncRoles([]);

$user = User::find(1);

// Get all roles (Eloquent relationship)
$roles = $user->roles;

$roleNames = $user->getRoleNames();

// Get role slugs as Collection
$roleSlugs = $user->getRoleSlugs();

Checking Roles & Permissions

Check Roles

$user = User::find(1);

// Check single role
if ($user->hasRole('admin')) {
    // User is admin
}

// Check by name
if ($user->hasRole('Administrator')) {
    // User is admin
}

// Check multiple roles (user has ANY of these)
if ($user->hasRole(['admin', 'editor'])) {
    // User has admin OR editor
}

// Using pipe-separated string
if ($user->hasRole('admin|editor|author')) {
    // User has admin OR editor OR author
}

// Using comma-separated string
if ($user->hasRole('admin,editor,author')) {
    // User has admin OR editor OR author
}

// Check ANY role (explicit method)
if ($user->hasAnyRole('admin', 'editor', 'author')) {
    // User has at least one of these roles
}

// Check ALL roles
if ($user->hasAllRoles('editor', 'author')) {
    // User has BOTH editor AND author roles
}

// Check EXACT roles (no more, no less)
if ($user->hasExactRoles('editor', 'author')) {
    // User has exactly these roles
}

// Check with specific guard
if ($user->hasRole('admin', 'api')) {
    // User has admin role in 'api' guard
}

Check Permissions

$user = User::find(1);

// Check single permission
if ($user->hasPermission('posts.create')) {
    // User can create posts
}

// Check ANY permission
if ($user->hasAnyPermission('posts.create', 'posts.edit', 'posts.delete')) {
    // User has at least one of these permissions
}

// Check ALL permissions
if ($user->hasAllPermissions('posts.create', 'posts.edit')) {
    // User has ALL of these permissions
}

// Using can() method (Laravel Gate compatible)
if ($user->can('posts.create')) {
    // User can create posts
}

Get User Permissions

$user = User::find(1);

// Get all permissions (from all roles)
$permissions = $user->getAllPermissions();

// Get permission names
$permissionNames = $user->getPermissionNames();
// Returns: Collection(['Create Posts', 'Edit Posts'])

// Get permission slugs
$permissionSlugs = $user->getPermissionSlugs();
// Returns: Collection(['posts.create', 'posts.edit'])

// Get permissions grouped by group
$grouped = $user->getGroupedPermissions();
// Returns:
// [
//     'Post Management' => Collection([Permission, Permission, ...]),
//     'User Management' => Collection([Permission, Permission, ...]),
//     'Uncategorized' => Collection([Permission, ...]),
// ]

Middleware Usage

Basic Syntax

// routes/web.php or routes/api.php

// Single role
Route::get('/admin', [AdminController::class, 'index'])
    ->middleware('role:admin');

// Single permission
Route::get('/posts/create', [PostController::class, 'create'])
    ->middleware('permission:posts.create');

Multiple Roles/Permissions

// Comma-separated (user needs ANY of these)
Route::get('/dashboard', [DashboardController::class, 'index'])
    ->middleware('role:admin,editor,manager');

// Pipe-separated (user needs ANY of these)
Route::get('/dashboard', [DashboardController::class, 'index'])
    ->middleware('role:admin|editor|manager');

// Multiple permissions
Route::get('/posts', [PostController::class, 'index'])
    ->middleware('permission:posts.view,posts.list');

Role OR Permission

// User needs EITHER the role OR the permission
Route::get('/reports', [ReportController::class, 'index'])
    ->middleware('role_or_permission:admin,reports.view');

With Guard

// Specify guard in middleware
Route::get('/api/admin', [ApiAdminController::class, 'index'])
    ->middleware('role:admin,guard:api');

// Permission with guard
Route::get('/api/posts', [ApiPostController::class, 'index'])
    ->middleware('permission:posts.view,guard:api');

Route Groups

// Apply to route group
Route::middleware(['auth', 'role:admin'])->group(function () {
    Route::get('/admin/dashboard', [AdminController::class, 'dashboard']);
    Route::get('/admin/users', [AdminController::class, 'users']);
    Route::get('/admin/settings', [AdminController::class, 'settings']);
});

// Nested middleware
Route::middleware(['auth'])->group(function () {
    
    // Admin routes
    Route::middleware(['role:admin'])->prefix('admin')->group(function () {
        Route::get('/dashboard', [AdminController::class, 'dashboard']);
        
        // User management (needs additional permission)
        Route::middleware(['permission:users.manage'])->group(function () {
            Route::resource('users', UserController::class);
        });
    });
    
    // Editor routes
    Route::middleware(['role:editor|author'])->prefix('content')->group(function () {
        Route::resource('posts', PostController::class);
    });
});

Controller Middleware

<?php

namespace App\Http\Controllers;

class PostController extends Controller
{
    public function __construct()
    {
        // Apply to all methods
        $this->middleware('auth');
        
        // Apply to specific methods
        $this->middleware('permission:posts.view')->only(['index', 'show']);
        $this->middleware('permission:posts.create')->only(['create', 'store']);
        $this->middleware('permission:posts.edit')->only(['edit', 'update']);
        $this->middleware('permission:posts.delete')->only('destroy');
    }
}

Middleware Syntax Reference

Syntax Description
role:admin User must have 'admin' role
role:admin,editor User must have 'admin' OR 'editor' role
role:admin|editor User must have 'admin' OR 'editor' role
role:admin,guard:api Check 'admin' role in 'api' guard
permission:posts.create User must have 'posts.create' permission
role_or_permission:admin,posts.edit User must have 'admin' role OR 'posts.edit' permission

Blade Directives

Role Directives

{{-- Check single role --}}
@role('admin')
    <a href="/admin">Admin Panel</a>
@endrole

{{-- With else --}}
@role('admin')
    <a href="/admin">Admin Panel</a>
@else
    <p>You don't have admin access.</p>
@endrole

{{-- Check multiple roles (any) --}}
@hasrole('admin|editor')
    <p>You are admin or editor</p>
@endhasrole

{{-- Check any role --}}
@hasanyrole('admin|editor|author')
    <p>You have at least one of these roles</p>
@endhasanyrole

{{-- Check all roles --}}
@hasallroles('editor|author')
    <p>You are both editor AND author</p>
@endhasallroles

{{-- Unless has role --}}
@unlessrole('admin')
    <p>You are NOT an admin</p>
@endunlessrole

Permission Directives

{{-- Check single permission --}}
@permission('posts.create')
    <a href="/posts/create" class="btn btn-primary">Create New Post</a>
@endpermission

{{-- With else --}}
@haspermission('posts.delete')
    <button class="btn btn-danger">Delete Post</button>
@else
    <button class="btn btn-danger" disabled>Delete Post</button>
@endhaspermission

{{-- Check any permission --}}
@hasanypermission('posts.edit|posts.delete')
    <div class="dropdown">
        <button class="btn btn-secondary dropdown-toggle">Actions</button>
        <div class="dropdown-menu">
            @haspermission('posts.edit')
                <a class="dropdown-item" href="#">Edit</a>
            @endhaspermission
            @haspermission('posts.delete')
                <a class="dropdown-item text-danger" href="#">Delete</a>
            @endhaspermission
        </div>
    </div>
@endhasanypermission

{{-- Check all permissions --}}
@hasallpermissions('posts.create|posts.edit|posts.delete')
    <p>You have full post management access</p>
@endhasallpermissions

Super Admin Directive

@superadmin
    <div class="alert alert-info">
        <i class="fas fa-shield-alt"></i>
        You have super admin access - all permissions are granted!
    </div>
@endsuperadmin

Guard Support

Understanding Guards

Guards determine how users are authenticated. OmniAccess supports multiple guards, allowing different authentication methods for different parts of your application.

Guard Priority

When checking roles/permissions, OmniAccess uses this priority:

  1. Guard specified in method call or middleware
  2. Guard from user model ($user->guard_name)
  3. Config default (config('omni-access.guards.default'))
  4. Laravel's default guard (auth.defaults.guard)

Setting User Guard

// In User model
class User extends Authenticatable
{
    use HasRoles;
    
    protected string $guard_name = 'web';
}

// API User model (different guard)
class ApiUser extends Authenticatable
{
    use HasRoles;
    
    protected string $guard_name = 'api';
}

Creating Roles for Specific Guards

// Web admin role
$webAdmin = Role::create([
    'name' => 'Web Admin',
    'slug' => 'web-admin',
    'guard_name' => 'web',
]);

// API admin role
$apiAdmin = Role::create([
    'name' => 'API Admin',
    'slug' => 'api-admin',
    'guard_name' => 'api',
]);

Checking Roles with Guard

$user = User::find(1);

// Check role in specific guard
if ($user->hasRole('api-admin', 'api')) {
    // User has api-admin role in 'api' guard
}

Middleware with Guard

// Check role in specific guard
Route::middleware('role:admin,guard:api')
    ->get('/api/admin', [ApiAdminController::class, 'index']);

Caching

How Caching Works

OmniAccess automatically caches:

  • User's roles
  • User's permissions
  • All roles list
  • All permissions list
  • All groups list
  • Individual role/permission lookups

Cache is automatically invalidated when:

  • Role is created, updated, or deleted
  • Permission is created, updated, or deleted
  • User's roles are changed
  • Role's permissions are changed

Configuration

'cache' => [
    'enabled' => env('OMNI_ACCESS_CACHE_ENABLED', true),
    'expiration_time' => env('OMNI_ACCESS_CACHE_TTL', 60 * 24),
    'key_prefix' => 'omni_access',
    'store' => env('OMNI_ACCESS_CACHE_STORE', null),
],

Manual Cache Management

use RbacSuite\OmniAccess\Services\CacheService;

$cache = app(CacheService::class);

// Forget specific user's cache
$cache->forgetUser($userId);

// Forget specific role cache
$cache->forgetRole($roleId);

// Forget all roles cache
$cache->forgetRoles();

// Forget all permissions cache
$cache->forgetPermissions();

// Flush ALL OmniAccess cache
$cache->flush();

User Cache Methods

$user = User::find(1);

// Clear and rebuild user's cache
$user->refreshPermissionsCache();

Disabling Cache for Development

# .env
OMNI_ACCESS_CACHE_ENABLED=false

Super Admin

Configuration

// config/omni-access.php

// Set to null to disable super admin functionality
'super_admin_role' => 'super-admin',

Creating Super Admin

// Create the super admin role
Role::create([
    'name' => 'Super Administrator',
    'slug' => 'super-admin',
    'description' => 'Full system access - bypasses all permission checks',
]);

// Assign to user
$user = User::find(1);
$user->assignRole('super-admin');

Super Admin Behavior

$superAdmin = User::find(1);
$superAdmin->assignRole('super-admin');

// Check if user is super admin
$superAdmin->isSuperAdmin(); // true

// ALL permission checks return true
$superAdmin->hasPermission('anything'); // true
$superAdmin->hasPermission('nonexistent.permission'); // true
$superAdmin->hasAnyPermission('foo', 'bar', 'baz'); // true
$superAdmin->hasAllPermissions('foo', 'bar', 'baz'); // true
$superAdmin->can('anything'); // true

// Super admin still respects role checks
$superAdmin->hasRole('editor'); // false (if not assigned)
$superAdmin->hasRole('super-admin'); // true

API Controller Example

<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use RbacSuite\OmniAccess\Models\Role;
use Illuminate\Http\Request;
use Illuminate\Http\JsonResponse;

class RbacController extends Controller
{
    public function __construct()
    {
        $this->middleware('permission:roles.view')->only(['roles', 'role']);
        $this->middleware('permission:roles.create')->only('createRole');
    }

    public function roles(): JsonResponse
    {
        $roles = Role::with('permissions')->get();
        
        return response()->json([
            'success' => true,
            'data' => $roles->map(fn ($role) => [
                'id' => $role->id,
                'name' => $role->name,
                'slug' => $role->slug,
                'permissions' => $role->permissions->pluck('slug'),
            ]),
        ]);
    }

    public function myPermissions(Request $request): JsonResponse
    {
        $user = $request->user();

        return response()->json([
            'success' => true,
            'data' => [
                'roles' => $user->getRoleSlugs(),
                'permissions' => $user->getPermissionSlugs(),
                'is_super_admin' => $user->isSuperAdmin(),
            ],
        ]);
    }
}

Database Seeder

Example Seeder

<?php

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use RbacSuite\OmniAccess\Models\Role;
use RbacSuite\OmniAccess\Models\Permission;
use RbacSuite\OmniAccess\Models\Group;
use App\Models\User;

class RbacSeeder extends Seeder
{
    public function run(): void
    {
        // Create Groups
        $groups = [
            ['name' => 'User Management', 'slug' => 'users', 'sort_order' => 1],
            ['name' => 'Role Management', 'slug' => 'roles', 'sort_order' => 2],
            ['name' => 'Post Management', 'slug' => 'posts', 'sort_order' => 3],
        ];

        foreach ($groups as $group) {
            Group::firstOrCreate(['slug' => $group['slug']], $group);
        }

        // Create Permissions
        $permissions = [
            ['name' => 'View Users', 'slug' => 'users.view', 'group' => 'users'],
            ['name' => 'Create Users', 'slug' => 'users.create', 'group' => 'users'],
            ['name' => 'Edit Users', 'slug' => 'users.edit', 'group' => 'users'],
            ['name' => 'Delete Users', 'slug' => 'users.delete', 'group' => 'users'],
            ['name' => 'View Posts', 'slug' => 'posts.view', 'group' => 'posts'],
                        ['name' => 'Create Posts', 'slug' => 'posts.create', 'group' => 'posts'],
            ['name' => 'Edit Posts', 'slug' => 'posts.edit', 'group' => 'posts'],
            ['name' => 'Delete Posts', 'slug' => 'posts.delete', 'group' => 'posts'],
        ];

        foreach ($permissions as $perm) {
            $group = Group::where('slug', $perm['group'])->first();
            
            Permission::firstOrCreate(
                ['slug' => $perm['slug']],
                [
                    'name' => $perm['name'],
                    'slug' => $perm['slug'],
                    'group_id' => $group?->id,
                ]
            );
        }

        // Create Roles
        $roles = [
            [
                'name' => 'Super Administrator',
                'slug' => 'super-admin',
                'description' => 'Full system access',
                'permissions' => [],
            ],
            [
                'name' => 'Administrator',
                'slug' => 'admin',
                'description' => 'Administrative access',
                'permissions' => [
                    'users.view', 'users.create', 'users.edit', 'users.delete',
                    'posts.view', 'posts.create', 'posts.edit', 'posts.delete',
                ],
            ],
            [
                'name' => 'Editor',
                'slug' => 'editor',
                'description' => 'Can manage posts',
                'permissions' => ['posts.view', 'posts.create', 'posts.edit'],
            ],
        ];

        foreach ($roles as $roleData) {
            $permissions = $roleData['permissions'];
            unset($roleData['permissions']);
            
            $role = Role::firstOrCreate(
                ['slug' => $roleData['slug']],
                $roleData
            );
            
            if (!empty($permissions)) {
                $role->syncPermissions($permissions);
            }
        }

        // Create Super Admin User
        $superAdmin = User::firstOrCreate(
            ['email' => 'admin@example.com'],
            [
                'name' => 'Super Admin',
                'password' => bcrypt('password'),
            ]
        );
        $superAdmin->assignRole('super-admin');

        $this->command->info('RBAC seeded successfully!');
    }
}

Testing

Feature Tests

<?php

namespace Tests\Feature;

use Tests\TestCase;
use App\Models\User;
use RbacSuite\OmniAccess\Models\Role;
use RbacSuite\OmniAccess\Models\Permission;
use Illuminate\Foundation\Testing\RefreshDatabase;

class RbacTest extends TestCase
{
    use RefreshDatabase;

    protected function setUp(): void
    {
        parent::setUp();
        
        // Clear cache before each test
        app(\RbacSuite\OmniAccess\Services\CacheService::class)->flush();
    }

    public function test_user_can_be_assigned_role(): void
    {
        $user = User::factory()->create();
        $role = Role::create(['name' => 'Admin', 'slug' => 'admin']);
        
        $user->assignRole('admin');
        
        $this->assertTrue($user->hasRole('admin'));
        $this->assertTrue($user->hasRole('Admin'));
    }

    public function test_user_can_have_multiple_roles(): void
    {
        $user = User::factory()->create();
        Role::create(['name' => 'Editor', 'slug' => 'editor']);
        Role::create(['name' => 'Author', 'slug' => 'author']);
        
        $user->assignRole('editor', 'author');
        
        $this->assertTrue($user->hasAllRoles('editor', 'author'));
        $this->assertCount(2, $user->roles);
    }

    public function test_user_has_permission_through_role(): void
    {
        $user = User::factory()->create();
        $role = Role::create(['name' => 'Editor', 'slug' => 'editor']);
        Permission::create(['name' => 'Create Posts', 'slug' => 'posts.create']);
        
        $role->givePermissionTo('posts.create');
        $user->assignRole('editor');
        
        $this->assertTrue($user->hasPermission('posts.create'));
    }

    public function test_super_admin_has_all_permissions(): void
    {
        $user = User::factory()->create();
        Role::create(['name' => 'Super Admin', 'slug' => 'super-admin']);
        Permission::create(['name' => 'Anything', 'slug' => 'anything.really']);
        
        $user->assignRole('super-admin');
        
        $this->assertTrue($user->isSuperAdmin());
        $this->assertTrue($user->hasPermission('anything.really'));
        $this->assertTrue($user->hasPermission('nonexistent.permission'));
    }

    public function test_role_middleware_blocks_unauthorized(): void
    {
        $user = User::factory()->create();
        
        $response = $this->actingAs($user)
            ->get('/admin');
        
        $response->assertStatus(403);
    }

    public function test_role_middleware_allows_authorized(): void
    {
        $user = User::factory()->create();
        Role::create(['name' => 'Admin', 'slug' => 'admin']);
        $user->assignRole('admin');
        
        $response = $this->actingAs($user)
            ->get('/admin');
        
        $response->assertStatus(200);
    }

    public function test_sync_roles_replaces_all_roles(): void
    {
        $user = User::factory()->create();
        Role::create(['name' => 'Admin', 'slug' => 'admin']);
        Role::create(['name' => 'Editor', 'slug' => 'editor']);
        Role::create(['name' => 'Author', 'slug' => 'author']);
        
        $user->assignRole('admin', 'editor');
        $this->assertCount(2, $user->roles);
        
        $user->syncRoles('author');
        $user->refresh();
        
        $this->assertCount(1, $user->roles);
        $this->assertTrue($user->hasRole('author'));
        $this->assertFalse($user->hasRole('admin'));
    }
}

Test Helper Trait

<?php

namespace Tests;

use RbacSuite\OmniAccess\Models\Role;
use RbacSuite\OmniAccess\Models\Permission;

trait RbacTestHelpers
{
    protected function createRole(string $name, ?string $slug = null): Role
    {
        return Role::create([
            'name' => $name,
            'slug' => $slug ?? \Str::slug($name),
        ]);
    }

    protected function createPermission(string $slug, ?string $name = null): Permission
    {
        return Permission::create([
            'name' => $name ?? ucwords(str_replace('.', ' ', $slug)),
            'slug' => $slug,
        ]);
    }

    protected function createUserWithRole(string $role): \App\Models\User
    {
        $user = \App\Models\User::factory()->create();
        $this->createRole(ucfirst($role), $role);
        $user->assignRole($role);
        return $user;
    }

    protected function createUserWithPermission(string $permission): \App\Models\User
    {
        $role = $this->createRole('Test Role');
        $perm = $this->createPermission($permission);
        $role->givePermissionTo($perm);
        
        $user = \App\Models\User::factory()->create();
        $user->assignRole($role);
        return $user;
    }
}

Troubleshooting

Common Issues

1. "User model must use HasRoles trait"

Problem: Middleware throws error about missing trait. 
// Solution: Add HasRoles trait to User model
use RbacSuite\OmniAccess\Traits\HasRoles;

class User extends Authenticatable
{
    use HasRoles; // Add this trait
}

2. Permissions not updating after role change

Problem: User still has old permissions after role modification. 
// Solution: Clear user's cache
$user->refreshPermissionsCache();

// Or clear all cache via artisan
php artisan omni:cache-reset

3. Middleware not working

Problem: role or permission middleware not found. 
// Solution 1: Check config - make sure register is true
'middleware' => [
    'register' => true,
],

// Solution 2: Manually register in app/Http/Kernel.php
protected $middlewareAliases = [
    'role' => \RbacSuite\OmniAccess\Middleware\RoleMiddleware::class,
    'permission' => \RbacSuite\OmniAccess\Middleware\PermissionMiddleware::class,
    'role_or_permission' => \RbacSuite\OmniAccess\Middleware\RoleOrPermissionMiddleware::class,
];

4. UUID primary keys not working

Problem: UUID columns not auto-generating. 
// Solution: Ensure config is correct and run fresh migrations
'primary_key_type' => 'uuid',
'user_primary_key_type' => 'uuid',

// Then run:
php artisan migrate:fresh

5. Cache not clearing properly

Problem: File cache doesn't support flush() properly. 
// Solution: Use Redis or manually clear specific keys
'cache' => [
    'store' => 'redis',
],

// Or manually clear:
$cache = app(CacheService::class);
$cache->forgetUser($userId);
$cache->forgetRoles();
$cache->forgetPermissions();

6. Guard mismatch errors

Problem: Role has different guard than user. 
// Solution: Ensure guards match or disable strict checking
$role = Role::create([
    'name' => 'API Admin',
    'guard_name' => 'api', // Must match user's guard
]);

// Or disable strict guard checking in config
'middleware' => [
    'strict_guard' => false,
],

Quick Reference

User Methods

Method Description
$user->assignRole('admin') Assign role(s) to user
$user->removeRole('admin') Remove role(s) from user
$user->syncRoles('admin', 'editor') Replace all roles
$user->hasRole('admin') Check if user has role
$user->hasAnyRole('admin', 'editor') Check if user has any role
$user->hasAllRoles('editor', 'author') Check if user has all roles
$user->hasExactRoles('editor') Check if user has exactly these roles
$user->hasPermission('posts.create') Check if user has permission
$user->hasAnyPermission(...) Check any permission
$user->hasAllPermissions(...) Check all permissions
$user->can('posts.create') Laravel Gate compatible check
$user->isSuperAdmin() Check if user is super admin
$user->getRoleNames() Get Collection of role names
$user->getRoleSlugs() Get Collection of role slugs
$user->getPermissionNames() Get Collection of permission names
$user->getPermissionSlugs() Get Collection of permission slugs
$user->getAllPermissions() Get all permissions
$user->getGroupedPermissions() Get permissions grouped by group
$user->refreshPermissionsCache() Clear and rebuild cache

Role Methods

Method Description
$role->givePermissionTo('posts.create') Assign permission(s) to role
$role->revokePermissionTo('posts.create') Remove permission from role
$role->syncPermissions(...) Replace all permissions
$role->hasPermission('posts.create') Check if role has permission
$role->getCachedPermissions() Get all role permissions (cached)
Role::findBySlugCached('admin') Find role by slug (cached)
Role::getAllCached() Get all roles (cached)
Role::getDefaultRole() Get default role
